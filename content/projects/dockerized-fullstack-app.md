## The Container Sandbox

A practical lab implementing micro-container architectures. It details best practices for creating optimized Dockerfiles and linking frontend services, APIs, and databases together.

### Key Aspects

* **Multi-Stage Builds**: Decreasing production image footprints by 70% using multi-stage compilations.
* **Volume Persistence**: Persisting raw database volumes locally during container reboots.
* **Network Isolation**: Configuring custom Docker networks to isolate the database layer from public routing.
